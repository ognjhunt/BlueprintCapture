# iOS City Launch Readiness Gates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the iOS city launch path fail-closed on real release, city, capture, pipeline, Meta-glasses, payout, and ops evidence instead of relying on stale notes or hard-coded assumptions.

**Architecture:** Keep the app capture-first. Fix the raw-bundle finalization order in Swift, remove local city-support fallback truth, and add a repo-local launch-readiness validator that verifies release config plus a proof JSON artifact for external/live facts that cannot be truthfully generated by code alone.

**Tech Stack:** Swift, Swift Testing, Bash, Python 3 standard library, Xcode build settings, Firebase/Blueprint HTTP routes.
